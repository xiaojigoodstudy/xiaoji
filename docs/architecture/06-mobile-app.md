# 06 - 移动 App 框架

## 1. 目标与边界
- 提供统一移动体验，补齐微信外场景。
- 支持更强提醒能力（本地推送/系统通知）。

## 2. 技术路线
- 推荐优先：`uni-app`（多端复用高）
- 可选：Flutter（性能与一致性更强）
- 一期默认采用 uni-app，后续按团队能力评估。

## 3. 功能模块
- 首页：今日重点提醒。
- 雷达：关注列表与详情。
- 打卡：今日任务、连续记录。
- 我的：账号、偏好、通知设置。

## 4. 推送策略
- 服务端触发 `notification` 后，发送 App Push。
- App 内维护通知中心，支持已读/未读。

## 5. MVP 验收标准
- 与微信端保持核心数据一致。
- 通知点击后可正确路由到业务页面。
