# 08 - 事件总线与规则引擎

## 1. 目标与边界
- 用事件解耦抓取、判定、发送三个阶段。
- 规则引擎只负责“是否触发”，不负责发送实现。

## 2. 事件主题（一期）
- `radar.item.discovered`
- `rule.trigger.requested`
- `notification.send.requested`
- `habit.checkin.completed`

## 3. 规则模型
- 维度：来源、标签、关键词、时间窗、优先级。
- 操作：包含、不包含、阈值比较、布尔组合。
- 输出：提醒模板、渠道、频率限制。

## 4. 可靠性设计
- 事件幂等消费（`eventId` 去重）。
- 失败重试 + 死信队列。
- 可回放最近事件用于排障。

## 5. MVP 验收标准
- 新抓取内容可稳定触发规则判定。
- 规则变更后在约定时间内生效。
